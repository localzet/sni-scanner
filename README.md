# SNI Scanner

Android-приложение для проверки, какие SNI/домены проходят до конкретных IP-адресов. Основной сценарий - подобрать рабочие пары `IP + SNI` для дальнейшей настройки Xray/VLESS в условиях фильтрации по whitelist.

Приложение собирает сетевые признаки: TCP-доступность, TLS handshake с SNI, HTTP-ответ, DNS-расхождения, базовую информацию по сертификату и гео/ASN.

## Что умеет

- Проверять один SNI на одном или нескольких IP.
- Проверять встроенный whitelist SNI на одном или нескольких IP.
- Проверять свой список SNI из поля ввода или `.txt` файла.
- Строить матрицу проверок `IP x SNI`.
- Запускать последовательную или параллельную проверку.
- Показывать сводку по IP: сколько SNI сработало и какой SNI лучший по задержке.
- Экспортировать полный результат в CSV.
- Продолжать долгую проверку с foreground-уведомлением.

## Режимы

`Single`  
Один SNI. Если IP один, показывается подробный отчет по этой паре. Если IP несколько, приложение проверяет этот SNI на всех IP и показывает общую таблицу.

`Whitelist`  
Берет список SNI из локального кеша whitelist. При первом запуске приложение предлагает скачать список. После этого проверки могут стартовать из кеша, а обновление выполняется отдельно.

`Massive`  
Свой список SNI. Можно вставить руками или загрузить `.txt`: один домен на строку, пустые строки игнорируются, `#` используется как комментарий.

## Ввод нескольких IP

Поле IP принимает один или несколько адресов. Можно разделять строками, пробелами, запятыми или `;`.

Пример:

```text
1.1.1.1
8.8.8.8, 9.9.9.9
203.0.113.10 # my server
```

Для каждого IP будет проверен каждый SNI из выбранного режима.

## Как читать результат

Основные статусы:

- `OK` - TLS с указанным SNI прошел. Обычно это главный кандидат для дальнейшей проверки в Xray/VLESS.
- `TLS_FAIL` - TCP до IP есть, но TLS с этим SNI не прошел. Может быть несовместимость, неверный SNI, фильтрация или особенности сервера.
- `BLOCKED` - TCP 443 не открылся или соединение оборвалось/истекло по таймауту.

В подробном отчете дополнительно есть:

- открытые TCP-порты;
- время TCP/TLS/HTTP;
- версия TLS и cipher suite;
- данные сертификата и SAN;
- сравнение DNS-ответов системного резолвера, Google DoH и Cloudflare DoH;
- эвристика SNI-vs-no-SNI для грубой оценки DPI-фильтрации;
- GeoIP/ASN по IP.

Это диагностический инструмент, а не абсолютный детектор блокировок. Один удачный или неудачный прогон не гарантирует стабильность: мобильные сети, DPI и CDN могут вести себя по-разному по времени, региону и оператору.

## CSV

CSV содержит все поля `CheckResult`, включая `ip`, `sni`, TCP/TLS/HTTP/DNS признаки, сертификат, GeoIP и ошибки. Для отбора кандидатов обычно достаточно отсортировать строки по:

1. `tlsOk = true`
2. минимальный `rttMs`
3. `httpOk = true`, если нужен именно HTTP-ответ
4. отсутствие подозрительных ошибок в `errors`

## Сборка

Требования:

- JDK 21
- Android SDK
- Gradle wrapper из репозитория

Debug APK:

```bash
./gradlew :app:assembleDebug
```

Unit-тесты:

```bash
./gradlew :app:testDebugUnitTest
```

На Windows:

```powershell
.\gradlew.bat :app:assembleDebug
.\gradlew.bat :app:testDebugUnitTest
```

Готовый debug APK после локальной сборки:

```text
app/build/outputs/apk/debug/app-debug.apk
```

## Структура

```text
app/src/main/java/com/localzet/sniscanner/
  MainActivity.kt              UI, режимы, прогресс, отображение результата
  WhitelistChecker.kt          сетевые проверки TCP/TLS/HTTP/DNS
  CheckResult.kt               модель результата
  CheckTarget.kt               пара IP + SNI
  ScanInputParser.kt           парсинг IP/SNI и построение матрицы целей
  ResultAggregator.kt          сортировка и сводки по результатам
  IpGeoInfo.kt                 GeoIP/ASN lookup
  WhitelistCache.kt            загрузка и кеш whitelist
  CsvExporter.kt               CSV-экспорт
  CheckForegroundService.kt    foreground-уведомление для долгих проверок
```

## Ограничения

- Долгие проверки пока оркестрируются из `MainActivity`; foreground service держит процесс в foreground, но не является полноценным worker-слоем.
- GeoIP и DoH-запросы зависят от внешних сервисов и могут ловить rate limit.
- Fast mode быстрее, но создает больше сетевой нагрузки и может давать менее стабильную картину.
- Debug APK не предназначен для публикации в сторы.
